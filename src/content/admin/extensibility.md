# Saltus 特進 拡張性・ロードマップ

> **対象読者:** システム管理者（ADMIN）のみ  
> 初稿完了後の機能拡張候補と、技術的な拡張方針をまとめたドキュメントです。

---

## 拡張方針（基本）

| 方針 | 内容 |
|------|------|
| 段階的リリース | Phase 単位で小さく追加し、本番影響を抑える |
| ロールファースト | 新機能は必ず MEMBER / MANAGER / ADMIN の権限表を先に定義 |
| Markdown ドキュメント | 使い方・仕様書と同様、コード外 Markdown で運用文書を管理 |
| DB 後方互換 | 既存データを壊さない追加（nullable カラム、論理削除の継続） |

---

## Phase 2 — 運用改善（優先度：高）

すぐ現場の運用負荷を下げる機能群です。

### 1. 通知

| 項目 | 内容 |
|------|------|
| 目的 | 返信・ステータス変更・宿題期限を見逃さない |
| 対象イベント | 問い合わせ返信、ステータス変更、MTG コメント、宿題期限前 |
| 実装案 | まずメール（Resend / SendGrid）、将来アプリ内通知テーブル |
| DB 追加案 | `Notification`（userId, type, readAt, payload Json） |

### 2. MTG 検索・整理強化

| 項目 | 内容 |
|------|------|
| タグ / カテゴリ | 例：進路・学習・メンタル |
| お気に入り | ユーザー単位のピン留め |
| 期間フィルタ | 学期・四半期単位 |
| DB 追加案 | `Tag`, `MeetingTag`, `MeetingFavorite` |

### 3. 宿題・フォローアップ管理

| 項目 | 内容 |
|------|------|
| 期限・担当 | homework JSON に `dueDate`, `assignee` を拡張 |
| ダッシュボード | 「未完了宿題一覧」ウィジェット |
| リマインド | 通知と連動 |
| 影響 | `MeetingDetailView`、一覧 API、ダッシュボード |

### 4. 問い合わせ運用改善

| 項目 | 内容 |
|------|------|
| 担当者アサイン | ADMIN が `assignedToUserId` を設定 |
| 内部メモ | ユーザー非表示の管理者コメント |
| FAQ 連携 | よくある問い合わせ → 使い方ガイドへリンク |
| DB 追加案 | Ticket に `assignedToUserId`, `TicketInternalNote` テーブル |

---

## Phase 3 — 管理・分析（優先度：中）

クラス運営・意思決定を支援する機能です。

### 5. メンバー・クラス管理

| 項目 | 内容 |
|------|------|
| 複数クラス | `Class` / `ClassMember` でマルチテナント化 |
| プロフィール | 学年、担当講師、入学年度 |
| 担当紐付け | MANAGER ↔ MEMBER の担当関係 |
| 影響 | **大** — 全クエリに `classId` スコープが必要 |

### 6. レポート・可視化

| 項目 | 内容 |
|------|------|
| KPI | MTG 回数、問い合わせ件数、平均解決日数 |
| 管理者ダッシュボード | グラフ + 期間指定 |
| エクスポート | CSV（MTG / 問い合わせ） |
| 実装案 | 集計 API + 管理画面タブ |

### 7. 監査ログ

| 項目 | 内容 |
|------|------|
| 記録対象 | 権限変更、削除/復旧、ステータス変更 |
| DB 追加案 | `AuditLog`（actorId, action, targetType, targetId, metadata Json） |
| 閲覧 | ADMIN 専用 `/dashboard/admin/audit` |

---

## Phase 4 — スケール・連携（優先度：中〜低）

中長期の拡張です。

### 8. ファイル添付

| 項目 | 内容 |
|------|------|
| 用途 | MTG 資料、問い合わせスクショ |
| ストレージ | Supabase Storage / S3 |
| DB 追加案 | `Attachment`（storageKey, mimeType, size, parentType, parentId） |

### 9. 外部連携

| 連携先 | 用途 |
|--------|------|
| Google Calendar | MTG 日付の同期 |
| Slack / LINE | 通知 Webhook |
| Google SSO | Workspace アカウントでログイン |

### 10. AI 支援（任意）

| 項目 | 内容 |
|------|------|
| MTG 要約 | 議事録から要点・宿題自動抽出 |
| 使い方 Bot | ガイドベースの Q&A |
| 注意 | 個人情報・API コスト・幻覚対策が必須 |

## 技術的拡張の土台

| 項目 | 理由 | 実装タイミング |
|------|------|----------------|
| **マルチテナント（classId）** | 複数特進クラス・年度で使い回す | Phase 3 前に設計 |
| **Feature Flag** | 機能の段階公開 | Phase 2 から検討可 |
| **公開 API 整備** | モバイル・外部連携 | Phase 4 |
| **定期バックアップ** | Supabase PITR / エクスポート | 運用開始と同時 |
| **E2E テスト** | 権限回りの退行防止 | Phase 2 以降 |

## 推奨ロードマップ（まとめ）

```
初稿（完了）
  ├─ MTG / 問い合わせ / 権限 / 使い方 / 管理 / 仕様書
  │
Phase 2（運用改善）
  ├─ 通知
  ├─ 宿題フォロー
  └─ 問い合わせ担当者・内部メモ
  │
Phase 3（管理・分析）
  ├─ レポート
  ├─ 監査ログ
  └─ クラス/メンバー管理（設計先行）
  │
Phase 4（スケール）
  ├─ ファイル添付
  ├─ 外部連携
  └─ AI（任意）
```

## 新機能追加時のチェックリスト

1. **権限表** — MEMBER / MANAGER / ADMIN の操作可否を定義
2. **Prisma スキーマ** — マイグレーション or `db push`、インデックス設計
3. **API** — Zod バリデーション、認可関数、エラーメッセージ
4. **UI** — サイドバー要否、FAB 影響、空状態メッセージ
5. **ドキュメント** — `member-guide.md` / 本仕様書の更新
6. **Render 環境変数** — 新規キーがあれば `.env.example` も更新

| 追加候補 | 内容 |
|----------|------|
| 変更履歴 | 版ごとの diff メモ |
| シーケンス図 | 認証・問い合わせフロー |
| 非機能要件 | 可用性、同時接続、データ保持期間 |
| 障害対応 | ロールバック手順、連絡フロー |

必要に応じて `src/content/admin/` に Markdown を追加し、本ページのタブ構成を拡張してください。
